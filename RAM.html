Introduction[edit]

Whonix CLI is recommended for advanced users only. However, it is possible to reduce the RAM and VRAM (video RAM) assignment of Whonix-Gateway even in Whonix â„¢ with the Xfce graphical user interface (GUI) without requiring any other configuration changes such as package installation or configuration file modifications. It is important to note that more RAM is required for upgrades.

While Whonix-Gateway â„¢ can function with as little as 256 MB RAM, resource-intensive operations like upgrades can cause the virtual machine to freeze. [1] Assigning less than 512 MB RAM to a Whonix â„¢ VM will result in RAM Adjusted Desktop Starter launching the command line interface (CLI) only, without Xfce. There are several options available, as described below.

Whonix â„¢ RAM and VRAM Defaults[edit]

  • Whonix â„¢ Xfce:
    • Whonix-Gateway Xfce - 1280 MB RAM and 128 MB VRAM.
    • Whonix-Workstation â„¢ Xfce - 2048 MB RAM and 128 MB VRAM.
  • Whonix â„¢ CLI:
    • Whonix-Gateway CLI - 512 MB RAM and 16 MB VRAM.
    • Whonix-Workstation â„¢ CLI - 512 MB RAM and 16 MB VRAM.

Advice for Systems with Low RAM[edit]

Whonix-Gateway â„¢ CLI RAM Saving Mode[edit]

To run Whonix-Gateway after its initial first boot setup or after upgrades:

  1. Shutdown Whonix-Gateway.
  2. Assign less RAM to Whonix-Gateway: Virtual machine â†’ Menu â†’ Settings â†’ Adjust Memory slider to 256 â†’ Hit: OK
  3. Assign less VRAM to Whonix-Gateway: VirtualBox â†’ click a VM â†’ Settings â†’ Display â†’ Graphics Controller â†’ Adjust slider for Video Memory to 16 â†’ OK
  4. Restart Whonix-Gateway.
  5. Start Whonix-Workstation â„¢.

Whonix-Gateway is now serving Whonix-Workstation â„¢ but operates with less RAM.

Whonix-Gateway â„¢ Graphical Administration Mode[edit]

Follow these steps in order to upgrade Whonix â„¢ or change Tor settings:

  1. Shut down all VMs.
  2. Assign default RAM to Whonix-Gateway. (Increase Virtual Machine RAM)
  3. Assign default VRAM to Whonix-Gateway.
  4. Start Whonix-Gateway.
  5. The user will now have access to a graphical Whonix-Gateway with Xfce.
  6. Perform administrative tasks such as upgrades and/or change Tor settings.
  7. Shutdown Whonix-Gateway.
  8. Apply RAM Saving Mode as described above.

Done. The user upgraded or changed settings using a graphical Whonix-Gateway with Xfce.

Whonix-Workstation â„¢ CLI RAM Saving Mode[edit]

Whonix-Workstation â„¢ CLI can be run with as little as 256 MB RAM when the Xfce desktop environment is not used. However, resource-intensive operations like upgrades can leave the virtual machine in a frozen state. [1] Assign more RAM during upgrades; 450 MB RAM should be more than enough.

Whonix-Workstation â„¢ Xfce RAM Saving Mode[edit]

Whonix-Workstation â„¢ Xfce can be run with as little as 512 MB RAM, but resource-intensive operations like upgrades can leave the virtual machine in a frozen state. [1] Assign more RAM during upgrades; 1280 MB RAM should be more than enough. In this configuration, the VM might freeze due to low RAM, or some applications might have reduced performance - this is especially true for browsers (Tor Browser), particularly when too many open browser tabs are open at the same time.

Alternatives[edit]

  • Whonix-Gateway â„¢ Xfce RAM Saving Mode: Whonix-Gateway â„¢ Xfce can run with as little as 512 MB RAM, but resource intensive operations like upgrades can leave the virtual machine in a frozen state. [1] For upgrading, increase RAM/VRAM as described above for Whonix-Gateway â„¢ Graphical Administration Mode.
  • Whonix â„¢ with CLI: Advanced users who are comfortable utilizing Whonix â„¢ with CLI can permanently set this option.
  • Experiments: Users can experiment with RAM assignments lower than 256 MB; please share your results.
